Core Feature — Phase 3

Multi-Level Approval Workflow Engine

📤

1. Requester Submits Change Request

Form includes title, description, priority (Low / Medium / High / Critical) and optional attachment. Request enters Pending state.

🔵

2. Level 1 Approver Assigned (Auto / Admin)

Admin assigns an L1 approver. System sends email notification. Request moves to Under Review (L1). L1 can Approve, Reject, or Escalate to L2.

🟣

3. Escalation — Level 2 / Level 3 (New in Phase 3)

L1 escalation routes to L2. Each level logs to approval_levels; DB trigger writes to audit_logs. Status shown as Under Review (L2/L3).

✅

4. Final Decision — Approved or Rejected

Final approver approves or rejects with mandatory comment. Status → Approved or Rejected. Email sent + Realtime WS pushes to all dashboards.

📄

5. PDF Export — Audit Trail (New in Phase 3)

Requester or Admin exports the complete request lifecycle as PDF — all level decisions, comments, timestamps, and final outcome. Generated via jsPDF on the client.
